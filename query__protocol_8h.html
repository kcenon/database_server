Detailed Description

Query protocol message structures for database gateway.

Defines the message structures for communication between clients and the database gateway server. Supports serialization/deserialization using container_system.

Protocol Design:

  • Binary protocol using container_system for efficient serialization
  • Request/response model for query execution
  • Support for parameterized queries
  • Token-based authentication

Thread Safety

All message structs (message_header, auth_token, query_request, query_response, etc.) are plain data structures with no internal synchronization. Individual instances should not be shared across threads without external synchronization. Serialization and deserialization create independent copies and are safe to call concurrently on different instances.
